# Review Gate Checklist

完整评审流程与输出模板见：`execution-playbook.md`。

- [ ] 输入材料齐全（**analyze** + **plan** + 人类约束）
- [ ] 风险/可行性/安全/性能/兼容/可测性已覆盖
- [ ] 关键发现有证据、有影响、有阻塞性判定
- [ ] 必改项与建议项已分离
- [ ] 阻塞歧义已用 `AskQuestion` 澄清
- [ ] 如用户明示跳过 **review**：已补“风险清单 + 兼容影响 + 测试范围 + 未决项处理 + 跳过理由”
- [ ] 闸门结论明确：通过 / 有条件通过 / 不通过
