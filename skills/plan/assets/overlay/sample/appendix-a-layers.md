# 附录 A：模块分层（sample 脚手架）

> **模板文件**：复制到业务仓 `project-overlays/<your-name>/` 后按实际模块替换下表。

## 模块速查

| 改动内容 | 目标模块 |
|----------|----------|
| 通用工具、基础模型 | `CoreKit` |
| 领域逻辑、协议封装 | `DomainLayer` |
| UI、页面与交互 | `FeatureUI` |
| 应用壳、导航与集成 | `AppShell` |

## 原则

- 依赖单向：上层可依赖下层，下层不反向依赖业务模块。
- 先定落层再写代码；跨模块改动须在方案中写影响面与回归范围。
