# Analyze Handoff Checklist

当用户明确要求“需求包”或“交给 **plan**”时，按此清单补齐。
完整流程与澄清规则见：`execution-playbook.md`。

- [ ] 目标与边界清晰（In / Out）
- [ ] 验收口径可验证（Given / When / Then）
- [ ] 待确认项已完整列出并标记阻塞性
- [ ] 关键口径变更已完成覆盖并清理旧口径正文（仅保留最新有效口径）
- [ ] 已决待确认项已回填正文并从清单移除（定稿不留已决问题）
- [ ] 阻塞项已通过 `AskQuestion` 获取明确决策（若用户暂不回答，已标注“等待用户答复”）
- [ ] 输出文档路径或正文已给出
